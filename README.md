# LCH-Farrow - Multi-GPU FFT/IFFT Benchmark

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![C++](https://img.shields.io/badge/C++-17-blue.svg)](https://isocpp.org/)
[![OpenCL](https://img.shields.io/badge/OpenCL-2.0+-orange.svg)](https://www.khronos.org/opencl/)
[![Platform](https://img.shields.io/badge/platform-Linux-lightgrey.svg)](https://www.linux.org/)

Multi-GPU FFT/IFFT Benchmark на OpenCL для Ubuntu с поддержкой NVIDIA и AMD GPU. Реализует дробную задержку сигнала с интерполяцией Лагранжа и FFT/IFFT свёртку с опорным ЛЧМ сигналом.

## 🎯 Основные возможности

- ✅ **Дробная задержка сигнала** с интерполяцией Лагранжа 5-го порядка
- ✅ **Формирование матрицы с задержанными сигналами** на GPU
- ✅ **Опциональный вывод с GPU** для анализа или оставление на GPU для дальнейшей обработки
- ✅ **Поддержка NVIDIA и AMD GPU** через OpenCL
- ✅ **In-place обработка** для экономии памяти
- ✅ **Профилирование производительности** всех этапов
- ✅ **Кроссплатформенность** - единый код для разных GPU

## 📋 Требования

- **ОС**: Ubuntu Linux (18.04+)
- **Компилятор**: GCC 9+ или Clang 10+
- **CMake**: 3.20+
- **OpenCL**: SDK для NVIDIA или AMD
- **clFFT**: библиотека для FFT (опционально, рекомендуется)

### Установка зависимостей (Ubuntu)

```bash
# OpenCL для NVIDIA
sudo apt-get install nvidia-opencl-dev

# OpenCL для AMD (если нужно)
sudo apt-get install opencl-headers

# clFFT
sudo apt-get install libclfft-dev
```

## 🏗️ Архитектура

Проект использует трёхслойную архитектуру:

1. **C++ Приложение** (платформа-независимо)
   - SignalBuffer - управление данными
   - FilterBank - FIR коэффициенты и ЛЧМ сигнал
   - ProcessingPipeline - координация обработки
   - ProfilingEngine - профилирование

2. **GPU Абстракция** (виртуальный интерфейс)
   - IGPUBackend - абстрактный интерфейс
   - OpenCLBackend - реализация для OpenCL
   - GPUFactory - выбор оптимального GPU

3. **OpenCL Код** (специфичный для платформы)
   - kernel_fractional_delay.cl - дробная задержка
   - kernel_hadamard.cl - поэлементное умножение
   - clFFT - FFT преобразования

## 🔧 Основные компоненты

### SignalBuffer
Управление сигнальными данными (1-256 лучей, 100-1300000 отсчётов на луч).

### FilterBank
Хранение FIR коэффициентов и опорного ЛЧМ сигнала. Предвычисление опорной FFT.

### LagrangeMatrix
Работа с матрицей коэффициентов Лагранжа 48×5 для точной дробной задержки.

### ProcessingPipeline
Координация этапов обработки до формирования матрицы с задержанными сигналами:
1. H2D Transfer (загрузка данных на GPU)
2. Дробная задержка (формирование матрицы)
3. Опционально: D2H Transfer (вывод с GPU для анализа)

**Режимы работы**:
- По умолчанию: результат остаётся на GPU
- С флагом `--copy-to-host`: результат копируется на хост для анализа

### OpenCLBackend
Реализация GPU обработки через OpenCL с поддержкой:
- NVIDIA GPU (RTX 3060, RTX 2080 Ti, и др.)
- AMD GPU (через OpenCL)

## 🎓 Дробная задержка

Проект реализует дробную задержку сигнала с использованием **интерполяции Лагранжа 5-го порядка** до формирования матрицы с задержанными сигналами.

### Алгоритм:
- Используется матрица коэффициентов 48×5
- Разрешение: 1/48 шага дискретизации (2.083%)
- Интерполяция по 5 точкам: `[n-2, n-1, n, n+1, n+2]`
- Точность выше чем линейная интерполяция
- Результат: матрица с задержанными сигналами на GPU

### Режимы работы:
- **По умолчанию**: результат остаётся на GPU для дальнейшей обработки
- **С флагом `--copy-to-host`**: результат копируется на хост для анализа

### Отладка:
См. [Doc/DEBUG_FRACTIONAL_DELAY.md](Doc/DEBUG_FRACTIONAL_DELAY.md) для детального маршрута отладки.
