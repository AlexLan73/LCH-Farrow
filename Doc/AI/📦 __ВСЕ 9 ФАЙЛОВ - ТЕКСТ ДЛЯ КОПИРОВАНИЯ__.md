## Key Insights (Why this architecture works)

### 1. Memory Magic ✨
```

RTX 2080 Ti: 11 GB available
Your task: 5.3 GB needed (if done naively)

Solution: In-place processing
Result: Only 2.66 GB needed
Bonus: 5.3 GB of margin for future features!

```

### 2. Speed Optimization ⚡
```

FFT-based convolution vs FIR:
FIR: 1.3M × 240 = 312M operations → 4.2 sec
FFT: 1.3M × log(1.3M) ≈ 41M operations → 0.15 sec
Speedup: 28× fewer operations!

But you still need fractional delay (can't do in frequency domain)
So: Fractional delay → FFT convolution (sequential, optimal)

```

## Architecture Decisions Made For You

| Decision | Trade-off | Why This Way |
|----------|-----------|------------|
| **In-place memory** | Less readable, more careful debugging | Save 50% GPU memory (5.3→2.66 GB) |
| **FFT convolution** | Requires reference precomputation | 28× fewer compute operations |
| **Batch FFT (size=256)** | Need to reshape data | Process all beams in 1 kernel call |
| **Virtual backend** | Slight overhead (function pointers) | Add new GPU type without touching main code |
| **Precomputed reference FFT** | More careful state management | 60 ms once vs 15 seconds (256 times) |
| **CMake + VSCode** | Not "native" IDE, learning curve | Works on Windows + Linux + Mac |
| **Async H2D/D2H** | More complex synchronization | Hide transfer latency in compute time |

## Key Classes Overview

### SignalBuffer
- Loads/saves 256 × 1.3M complex samples
- Host pinned memory management
- Methods: LoadFromFile(), SaveToFile(), GetBeam(id)

### FilterBank
- 240 FIR coefficients (48 × 5)
- Reference signal storage
- Precomputes reference FFT (one-time!)
- Methods: LoadCoefficients(), PrecomputeReferenceFft()

### ProcessingPipeline
- Orchestrates: delay → FFT → multiply → IFFT
- GPU dispatch
- Handles synchronization
- Methods: ExecuteFull(), ProfileKernel()

### GPUFactory
- Auto-detects available GPUs
- Selects optimal device
- Returns CUDABackend or HIPBackend

## Performance Expectations

**RTX 2080 Ti:**
- Total E2E: 4.65 seconds
- H2D: 170 ms
- GPU kernel: 4200 ms (90% of total)
- FFT ops: 125 ms
- D2H: 170 ms

**RTX 3060:**
- Total E2E: 7.3 seconds (slower bandwidth 360 vs 616 GB/s)

## Critical Patterns

### In-Place Processing
```

// NOT: separate input/output buffers (uses 2× memory)
// YES: reuse same buffer throughout pipeline
d_buffer → kernel → d_buffer (still in-place)
d_buffer → FFT forward (in-place)
d_buffer → multiply (in-place)
d_buffer → FFT inverse (in-place)

```

### Batch FFT
```

// Batch size = 256 (all beams at once)
cufftPlan1d(\&plan, 1300000, CUFFT_C2C, 256);
cufftExecC2C(plan, d_buffer, d_buffer, CUFFT_FORWARD);
// One call processes all 256 beams!

```

### Reference FFT Precomputation
```

// Compute ONCE in initialization
FilterBank::PrecomputeReferenceFft();

// Then REUSE 256 times (cached on GPU)
for (int beam = 0; beam < 256; beam++) {
multiply_kernel(d_beam, d_reference_fft);
}

```

## Profiling Examples

```

# 🎮 MULTI-GPU STRATEGY: NVIDIA + AMD

## Virtual GPU Backend Pattern

### Abstract Interface (One file for all!)
```

class IGPUBackend {
virtual void ExecuteConvolution(
const complex<float>* input,
complex<float>* output,
const float* coefficients,
uint32_t num_beams,
uint32_t num_samples
) = 0;
virtual string GetBackendName() = 0;
};

```

### CUDA Backend (NVIDIA: RTX2080Ti + RTX3060)
```

class CUDABackend : public IGPUBackend {
void ExecuteConvolution() override {
// H2D transfer
kernel_fractional_delay<<<>>>();
cuFFT forward
kernel_hadamard<<<>>>();
cuFFT inverse
// D2H transfer
}
};

```

### HIP Backend (AMD: RX6900XT + MI300X) [Future]
```

class HIPBackend : public IGPUBackend {
void ExecuteConvolution() override {
// Same interface, different compute
kernel_convolution_rdna2();  // RX6900XT optimized
// or
kernel_convolution_cdna3();  // MI300X optimized
}
};

```

# 🚨 COMMON PITFALLS & SOLUTIONS

## Problem #1: CMake can't find CUDA

**Solution:**
```

cmake .. -DCUDA_TOOLKIT_ROOT_DIR="C:/Program Files/NVIDIA GPU Computing Toolkit/CUDA/v13.0"

```

## Problem #2: cuFFT linking error

**Fix in CMakeLists.txt:**
```

find_package(CUDAToolkit REQUIRED)
target_link_libraries(radar_convolver
CUDA::cufft         \# FFT library
CUDA::cudart        \# Runtime
)

```

## Problem #3: Out of GPU Memory

**Cause:** Not using in-place processing
**Solution:**
```

// Use SAME buffer throughout
d_buffer → kernel (writes to d_buffer)
d_buffer → FFT (in-place)
d_buffer → multiply (in-place)

```

## Problem #4: Pinned memory fails

**Fix:**
```

cudaMallocHost(\&host_input, size);    // Pinned
// ... use ...
cudaFreeHost(host_input);              // Free pinned

```

## Problem #5: cuFFT batch incorrect

**Fix:**
```

cufftPlan1d(\&plan, 1300000, CUFFT_C2C, 256);  // batch=256!
cufftExecC2C(plan, d_buffer, d_buffer, CUFFT_FORWARD);

```

## Problem #6: Reference FFT in loop (SLOW!)

**Cause:**
```

for (int b = 0; b < 256; b++) {
cufftExec(reference);  // 60ms × 256 = 15 sec! ❌
}

```

**Solution:**
```

cufftExec(reference);      // 60 ms, ONE TIME ✅
// Cache result, reuse 256 times

```

## Problem #7: Synchronization missing

**Fix:**
```

kernel<<<>>>();
cudaDeviceSynchronize();   // WAIT for kernel to finish
cudaMemcpy(...);           // Now safe to copy

```

## Problem #8: Memory leak detection

```

cuda-memcheck ./radar_convolver

```

## Problem #9: Kernel timeout (TDR)

**Windows Registry:**
```

HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\GraphicsDrivers
Key: TdrDelay (DWORD)
Value: 60  (seconds)

```

## Problem #10: VSCode red squiggles

**Fix .vscode/settings.json:**
```

{
"cmake.configureArgs": [
"-DCUDA_TOOLKIT_ROOT_DIR=C:/Program Files/NVIDIA GPU Computing Toolkit/CUDA/v13.0"
]
}

```

## Debug Table

| Error | Cause | Fix |
|-------|-------|-----|
| "undefined reference to cufftExec" | Missing CUDA::cufft link | Add to target_link_libraries |
| "Out of memory" | Separate input/output buffers | Use in-place processing |
| Results are garbage | Missing cudaDeviceSynchronize | Add after kernel |
| Program very slow (30 sec) | Reference FFT in loop | Precompute once |
| CUDA not found | Wrong toolkit path | Set CUDA_TOOLKIT_ROOT_DIR |
| Non-coalesced access | Strided memory reads | Linearize data layout |
```
